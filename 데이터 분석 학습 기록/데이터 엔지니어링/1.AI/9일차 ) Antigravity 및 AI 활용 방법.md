## **글 쓴 이유**

 - 9일차 부터는 데이터 분석이 아닌 데이터 엔지니어링 학습을 시작함

 - AI를 활용한 데이터 엔지니어링을 하기 위해 환경 구성을 진행

 - 환경 구성에 필요한 IDE와 AI 활용방법에 대해 정리하겠음

## **Antigravity ?**

 - **VS Code 기반으로 구축된 IDE로써 VS Code에서 접했던 익숙한 UI를 제공**

 - Google Gemini 모델이 에디터 전반에 통합되어 소스를 만들어주거나 , 상호작용이 가능함

 - 대중적으로 Cursor가 AI 코딩 도구로써 주도하던 분위기 였지만 Google에서 gemini 3 발표와 함께 공개하여 대체제로써 사용됨

### **주요 기능?**

 - 에이전트로써의 기능 수행 : 사용자의 명령을 받아 코드 작성 , 터미널 명령 실행 , 웹 브라우저 검색 등 스스로 수행 

 - 에디터로써의 기능 수행 : AI 기능이 탑재된 IDE로써 일반적인 코드 편집 기능 외에도 AI에게 바로 일을 시킬 수 있는 기능 포함

 - 아티팩트로써의 기능 수행 : AI가 작업을 수행 후 사용자에게 제출하는 결과물 , AI가 무엇을 만들었고 , 작업 진행 상황은 어떤지 눈으로 확인 가능

 - 에이전트 매니저로써의 기능 수행 : 여러 AI를 동시에 관리 할 수 있는 화면 제공

 - 지식과 피드백 제공 : 사용자와의 대화를 통해 AI가 학습하고 기억하는 기능 , 상호 피드백을 거쳐 원하는 결과물 도출

### **주요 특징?**

 - Agent Manager - 여러 작업을 동시에 관리 가능

   -> 여러 에이전트를 병렬로 실행 가능함 , 하나는 리펙토링 , 다른 하나는 테스트 코드 생성 , 다른 하나는 문서 작성으로 각각의 작업을 분할시켜 명령 할 수 있다.

 - Planning vs Fase 모드 지원

  -> Planning 모드는 코드를 짜기 전에 상세한 계획서를 먼저 보여줌 , 복잡한 프로젝트에 적합

  -> Fast 모드는 계획없이 바로 실행해서 간단한 수정에 유용함

 - 브라우저 통합 - Chrome에서 자동 UI 테스트

  -> 크롬 확장을 설치하면 에이전트가 직접 브라우저를 제어함 , Cursor에는 없는 기능

 - 다중 모델 지원 - Gemini 3 Pro , Claude 4.5등 지원

  -> 모델선택이 자유로워 원하는 작업별로 모델을 선택해서 작업을 명령 할 수 있다.

### **AI 사용 방법?**

 - 목표 설정 : AI에게 시킬 큰 작업을 정의 (예시 : 금일 날씨 데이터를 추출해서 지역별로 알려주는 앱을 만들어줘)

 - 작업 지시 : 에이전트가 에디터와 브라우저를 오가며 코드를 짜고 필요한 정보를 찾음

 - 결과 확인 : 에이전트가 만들어온 결과물을 확인

 - 피드백 및 수정 : 결과물이 마음에 들지 않거나 수정이 필요하면 댓글을 달듯 피드백을 줌

 - 완성 : AI가 피드백을 반영해서 결과물을 수정하고 , 최종적으로 원하는 프로그램을 완성함

## **AI를 활용하면 소스는 볼 필요 없다 ?**

 아니다. 요즘에는 개발자 , 비개발자 모두 AI에게 코딩 작업을 요청하면 AI가 스스로 해당 기능을 수행하는 코드를 짜주기 때문에 별도 소스 작성 지식이 없어도 가능하지만 해당 코드가 어떤걸 의미하고 , 어떤 로직을 통해 예외처리가 되고 수행하는지에 대한 해석 능력이 있어야한다.  그래야 AI가 정말로 내가 원하는 동작에 대한 코딩을 잘 짜주었는지 확인이 가능하며 , 확인 전 정말 원하는 동작이 맞는지도 파악 할 수 있기 때문이다. **소스코드 해석 능력이 필수역량이 되어야하는데 이걸 코드 문해력**이라고 한다.

### **코드 문해력**

 - 처음부터 코드를 작성하는 역량보다는 AI가 제안한 코드를 분석하고 오류를 찾아내는 능력이 실무 속도의 차이를 만듦

 - 사수가 없는 환경에서도 AI에게 코드의 라이변 설명을 요청해서 스스로 학습하는 능동적 태도 필요

 - 동일한 결과물을 내더라도 AI를 활용해 소요 시간을 획기적으로 단축하는 인재가 업계에서 높은 평가를 받음

#### **코드 문해력의 구성 요소는 ?**

 코드 문해력은 크게 세 가지 층위로 나뉜다.

 - 구문 이해 : 프로그래밍 언어의 문법과 규칙을 이해하고 키워드의 역할을 아는 단계

 - 논리 추론 : 조건문 , 반복문 , 함수 호출의 흐름을 따라가며 프로그램이 어떤 순서로 동작할지 머릿속으로 시뮬레이션하는 능력

 - 의도 파악 : 이 코드가 '왜 ?' 이렇게 작성되었고 , 비즈니스 로직과 설계 의도를 읽어내는 가장 우선시되는 단계의 능력

#### **코드 문해력이 왜 중요해 ?**

 - 유지보수의 효율성 : 기존 코드를 빠르게 이해해야 버그를 수정하거나 기능을 추가 할 수 있음

 - 협업 능력 : 동료(또는 AI)가 작성한 코드를 리뷰하고 피드백을 주고받는 과정의 기초가 됨 ,

 - 디버깅 속도 : 코드가 실행돠는 흐름을 정확히 읽을 수 있다면 에러가 발생한 지점을 찾는 시간이 줄어듬

 - 좋은 코딩 작성 : 남이 만들거나 AI가 생성한 코드를 잘 읽는 사람이 가독성 좋은 코드를 작성하거나 수정할 확률이 높음
