## **파이썬 자료형**

 - 단순 자료형 : int(정수)  , float(실수) , str(문자열) , bool(논리형)

 - 복합 자료형 : list(리스트) , tuple(튜플) , set(집합) , dict(딕셔너리)

|   |   |   |   |
|---|---|---|---|
|명칭|형식|형태|특징|
|list|[1,2,3]|대괄호|각 항목별로 인덱스가 있어 중복값 저장 가능|
|tuple|(1,2,3)|소괄호|각 항목별로 인덱스가 있어 중복값 저장 가능|
|set|{1,2,3}|중괄호|각 항목별로 인덱스가 없어 중복값 저장 불가능|
|dict|{1:A , 2:B , 3:C}|중괄호 , 키 : 값 쌍으로 이루어짐|인덱스가 없지만 대신 키로 판단하여 , 중복 키가 저장될시 나중에 저장된 키 : 값으로 저장됨|

## **파이썬 array**

 - 파이썬은 리스트 배열을 생성하기 위해 array 함수를 사용함

 - array를 사용하여 만든 리스트에는 정수 , 실수 , 문자열 타입이 다 저장 될 수 있음

 - type값이 통일되지 않아 데이터 저장시 불편함

 - 요소별 연산이 불가능함 , list1과 list2의 요소를 더하려고 list1 + list2를 하면 해당 요소가 합쳐져서 덧셈이 안됨

 ex ) list1 = [1,2,3] , list2 = [4,5,6] , list1+list2 = [1,2,3,4,5,6] => 더해지지가 않음

## **파이썬 numpy**

 - 위의 array의 단점을 극복하고 수치계산을 효율적으로 수행하기 위해 numpy 라이브러리가 도입됨

 - numpy는 리스트를 생성할때 type값을 모두 통일시켜버림

 ex ) list1 = [1, 2.0, 4] , arr1 = numpy.array(list1) , arr1 => [1. , 2. , 3. ]  형식으로 전부 실수로 변함

 - 하지만 해당 변환하는것이 데이터 저장시 여러 타입을 저장할 경우에 비효율적이라 pandas가 개발됨
